# Pico Scheduler Firmware Generator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the generic Pico `main.py` + `state.json` deployment with generated controller-specific `main.py` firmware that preserves the current host-side JSON source of truth.

**Architecture:** Add a host-side generator under `pico_scheduler/` that renders concrete MicroPython from validated scheduler state and options. Update the server apply path to generate a temporary `main.py`, copy only that file to the Pico, and accept the new `type: report` wire envelope while preserving host-side `devices` state handling.

**Tech Stack:** Python 3, FastAPI server, MicroPython firmware text generation, `unittest`, `mpremote`

---

```python
import json
from textwrap import indent


def _machine_import(types: set[str]) -> str:
    if types == {"pwm"}:
        return "from machine import Pin, PWM"
    return "from machine import Pin"


def _provenance_block(controller_id: str, state: dict, git_version: str, generated_at: str, options: GeneratorOptions) -> str:
    payload = {
        "report_every": state["report_every"],
        "devices": state["devices"],
        "options": {
            "loop_sleep_ms": options.loop_sleep_ms,
            "pwm_freq": options.pwm_freq,
        },
    }
    return '\n'.join(
        [
            '"""',
            'Generated by: plamp/pico-firmware/pico_scheduler_generator.py',
            f"Generated at: {generated_at}",
            f"Git version: {git_version}",
            f"Controller: {controller_id}",
            "Generator input:",
            json.dumps(payload, indent=2),
            '"""',
        ]
    )
```

Add matching helpers for:

- rendering `devices = [...]`
- rendering GPIO device setup lines like `Pin(15, Pin.OUT)`
- rendering PWM device setup lines like `PWM(Pin(16))`
- rendering `error()` and `report()` with `type`
- omitting PWM constants/imports when no PWM devices exist
